# Monetization

## Principle

Revenue comes from advertising and sponsorships, not courses, paid docs, paid API, or selling user requests.

## What We Do Not Pay For

We should not pay for users' AI usage.

Correct model:

- user installs package;
- user runs code inside their app;
- user uses their own OpenAI/Anthropic/Gemini key;
- provider bills user's account.

Our public website uses mock demos by default.

## Revenue Stages

### Stage 0: 0-50k pageviews/month

Focus:

- product quality;
- GitHub stars;
- npm downloads;
- SEO pages;
- newsletter subscribers.

Revenue expectation: low.

Do not over-optimize ads yet.

## Additional Monetization Models

Баннерная реклама не должна быть единственным источником дохода. Для developer hub лучше подходят форматы, которые совпадают с намерением пользователя.

### Sponsored integrations

Компании платят за полезную, честно помеченную интеграцию:

- AI chat with Supabase;
- AI chat with Vercel AI SDK;
- AI chat with Clerk auth;
- AI chat with PostHog analytics;
- AI chat with Langfuse tracing;
- AI chat with Pinecone, Upstash, Neon, Sentry, Trigger.dev.

Это дороже баннера и полезнее для читателя, если интеграция действительно рабочая.

### Affiliate and referral links

Можно использовать прозрачные referral links на инструменты, которые разработчику все равно нужны:

- hosting;
- auth;
- database;
- vector database;
- observability;
- analytics;
- LLM providers;
- error tracking.

Правило: помечать sponsored/referral links явно и не рекомендовать плохой продукт ради комиссии.

### Paid UI blocks and templates

Бесплатное ядро:

- hooks;
- базовые examples;
- docs;
- playgrounds.

Платные артефакты позже:

- polished AI chat UI block;
- agent timeline;
- RAG answer layout;
- file-analysis UI;
- voice assistant UI;
- multi-model playground;
- team/commercial license.

Это не курс и не paywall для документации. Это готовые production blocks для тех, кому важна скорость.

### GitHub Sponsors and Open Collective

Подходит как дополнительный канал доверия и поддержки open-source части, но не как основной бизнес-план.

### Job board and hiring sponsors

Позже, если появится стабильная аудитория React/AI developers:

- AI frontend jobs;
- sponsored job posts;
- company hiring pages.

## Preferred Revenue Mix

На практике стоит строить так:

1. Бесплатный developer hub и open-source пакет.
2. Affiliate/referral links там, где есть высокий intent.
3. Sponsored integrations после первых признаков трафика.
4. Developer ads как фон.
5. Paid UI blocks/templates, если examples хорошо заходят аудитории.

### Stage 1: 50k-250k pageviews/month

Possible:

- EthicalAds;
- small direct sponsorships;
- newsletter sponsorship tests.

Ad inventory:

- one tasteful docs ad;
- calculator sponsor;
- model table sponsor;
- newsletter sponsor.

### Stage 2: 250k-1M pageviews/month

Possible:

- Carbon Ads / BuySellAds conversations;
- direct sponsor packages;
- sponsored integrations;
- monthly sponsor bundle.

Good sponsor categories:

- AI model providers;
- observability;
- vector databases;
- hosting;
- auth;
- analytics;
- error tracking;
- databases;
- browser automation;
- CI/CD;
- devtools.

### Stage 3: 1M+ pageviews/month

Possible:

- direct ad sales;
- multiple monthly sponsors;
- premium homepage/tools placements;
- higher-value newsletter sponsorships.

## Ad Inventory Ideas

- Homepage sponsor.
- Docs sidebar sponsor.
- LLM cost calculator sponsor.
- Model comparison table sponsor.
- Streaming playground sponsor.
- Newsletter sponsor.
- "Provider page" sponsor.
- GitHub README sponsor, if acceptable.

## Guardrails

- Ads must not break trust.
- No deceptive native ads.
- No fake recommendations.
- No paywall for core content.
- No selling user data.
- No tracking-heavy ad setup early.

## Core Metrics

Audience:

- monthly pageviews;
- unique visitors;
- returning visitors;
- newsletter subscribers;
- GitHub stars;
- npm downloads.

Ad readiness:

- traffic by country;
- traffic by page type;
- top pages;
- CTR on internal tools;
- sponsor slot impressions.

Business:

- monthly ad revenue;
- revenue per 1000 pageviews;
- sponsor pipeline;
- renewal rate.
